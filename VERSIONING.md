# Versioning

AegisForge follows a simple release-oriented versioning approach.

## Format

Recommended format:

- `MAJOR.MINOR.PATCH`

Examples:

- `0.1.0`
- `0.1.1`
- `0.2.0`

## Guidance

### Major
Increase the major version when the public runtime contract changes in a breaking way.

### Minor
Increase the minor version when new features are added in a backward-compatible way.

### Patch
Increase the patch version for fixes, cleanup, and small non-breaking improvements.

## Release alignment

The version in the Agent Card, README, provenance metadata, and release notes should agree.
